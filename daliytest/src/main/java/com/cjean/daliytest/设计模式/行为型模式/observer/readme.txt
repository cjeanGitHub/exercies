设计模式之：
	行为型模式：
		观察者模式（广播机制）：
			聊天室、网游中的世界广播、cs中服务器需要将每个人的方位发送给需要的人
			可以使用观察者模式来处理。将多个订阅者、客户称之为观察者；需要同步给多个订阅者的数据
				封装到对象中，称之为目标
		核心：
			・ 观察者模式主要用于1：N的通知，当一个对象（目标对象Subject或Objservable）的状态发生改变时
				他需要及时通知一系列对象（观察者对象，objserver），令他们做出相应
		
		通知观察者的方式：
			push(推送模式)：
				・ 每次都会把通知一广播方式发送给所有观察者，所有观察者都只能被动接受
			
			pop（拉）：
				观察者只要知道有情况即可。至于什么时候获取内容，获取什么内容，都可以自主决定
		
		2.在jdk中一已经提供了观察者模式大部分的功能
			import java.util.Observable 对象继承类
			
			import java.util.Observer; 观察者实现类
			
			
		开发中常见的场景：
				- 聊天室
				- 网络游戏 客户端状态
				- servlet中关于监听器的实现
				- Andriod广播机制
