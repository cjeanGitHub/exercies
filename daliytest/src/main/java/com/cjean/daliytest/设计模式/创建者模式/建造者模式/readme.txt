建造者模式
	例子 ： JDk中StringBuilder就是建造者的模式   一般以Builder结尾的都是 建造者 模式
	
	场景：
		 需要建造一个特别负责的产品， 如飞船、Iphone 。这个负责的产品的创建，有这样一个问题需要处理
			- 装配这些子组件是不是有个步骤问题？
		实际开发过程中，我们需要对象构建是，也是非常复杂的，需要多个步骤进行处理
	
	建造者的本质：
		- 分离对象子组件的单独构造（有builder来负责构建子组件）和装配（由directer负责对象的构建）
			。从而建造出复杂的对象，这个模式适用于：某个对象的构建过程复杂的情况使用。
		- 由于实现了构建个装配的解耦， 不同的构建器，相同的装配，也可以做出不同的对象；
			相同的构建器，不同的装配顺序下，做出不同的对象。也是实现了构建算法，装配算法发解耦，
			达到了更好的复用。
