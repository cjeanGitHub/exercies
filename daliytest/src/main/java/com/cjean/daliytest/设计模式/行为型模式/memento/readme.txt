设计模式之：
	行为型模式：
		备忘录模式《保存目标状态内容的机制》：
			结构:
				源发器类
				备忘录类
				负责人类
		核心：
			・ 观察者模式主要用于1：N的通知，当一个对象（目标对象Subject或Objservable）的状态发生改变时
				他需要及时通知一系列对象（观察者对象，objserver），令他们做出相应
		
		备忘点多时：
			将备忘对象存储到硬盘中   时使对象序列化 持久化
		
			
			
		开发中常见的场景：
				- 棋类 悔棋
				- 普通软件的撤销操作
				- 数据库软件中 事务管理中的回滚操作
				- Photoshop中的 历史记录
