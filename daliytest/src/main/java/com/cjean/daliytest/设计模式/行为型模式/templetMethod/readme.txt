设计模式之：
	行为型模式：
		模板方法模式：
			对流程很了解（比如，银行办业务：1.取号、2.办理业务《业务不确定》，3.服务评价），但是其中流程存在不确定的因素，所以使用模板方法模式
			
		什么时候一到模板方法模式：
			 实现一个算法是，整体步骤都很固定。但是，某些部分容易发生改变。易变的部分可以抽象出来，供子类实现；
		开发中常见的场景：
			非常频繁。各种框架，类库中都有他的影子。比如常见的有：
				- 数据库访问的封装
				- Junit单元测试
				- servlet中关于doGet、doPost方法的调用
				- Hibernate中模板程序
				- spring中JDBCTemplate、HibernateTemplate等
		方法回调（钩子方法）
		 -好莱坞原则：我们会通知你
		 	- 在软件开发过程中，我们可以将call翻译为调用。子类不能调用父类，而通过父类调用子类。
		 		这些调用步骤已经在父类中写好了，完全由父类控制整个过程
