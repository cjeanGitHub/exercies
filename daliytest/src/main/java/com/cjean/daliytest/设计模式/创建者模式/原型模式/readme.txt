原型模式（克隆/拷贝模式）（prototype）
	场景：
		- 克隆技术的过程？克隆羊 多利
		- javascript语言中的继承是怎么实现的？那里也有prototype 
	
	原型模式：
		- 通过new产生一个对象的时间非常繁琐的数据准备或者访问权限，则可以使用原型模式
		- 就是java中的克隆技术，以某个对象为原型，克隆出新的对象。显然，新对象具备原型对象的特点
		- 优势：效率高（直接克隆），避免重新执行构造过程步骤
		- 克隆类似于new，但是不同与new。new创建新的对象属性采用的是默认值，克隆出来的对象的属性值完全
			和原型对象相同。并且克隆出的新对象的改变不会影响原型对象，然后在修改克隆对象的值
	原型模式实现原理：
		-Cloneable接口和clone方法
		- prototype 模式中实现起来最困难的地方就是内存复制操作，所幸java中提供clone方法替我们做了
			绝大部分的事情
